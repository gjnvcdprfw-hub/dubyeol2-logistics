# Railway 빠른 시연 배포 메모

이 문서는 정식 운영 배포가 아니라 빠른 시연 배포 기준이다.

## 목적

- GitHub에 올라간 `main` 브랜치를 Railway에서 바로 띄운다.
- 셀러 회원가입, 주문 요청, 관리자 입고/견적/예치금/포장 입력 흐름을 URL로 확인한다.
- 정식 운영용 DB, 파일 저장소, 도메인, 실계좌, 회사 정보는 아직 붙이지 않는다.

## Railway 설정

1. Railway에서 `gjnvcdprfw-hub/dubyeol2-logistics` GitHub repo를 연결한다.
2. Variables에 `SESSION_SECRET`을 추가한다.
   - 32자 이상 임의 문자열을 쓴다.
   - 이 값은 GitHub에 올리지 않는다.
3. 서비스가 배포되면 공개 URL을 생성한다.

## 시작 명령

`railway.json`은 아래 흐름으로 앱을 시작한다.

```bash
npm run start
```

`npm run start`는 시연용 SQLite DB에 migration을 적용한 뒤 Railway의 `PORT`로 Next.js를 띄운다.

## 관리자 계정 만들기

배포 후 Railway Shell에서 아래 명령을 한 번 실행한다.

```bash
npm run seed:admin -- <admin-email> <admin-password>
```

비밀번호는 8자 이상이어야 한다.

## 빠른 시연 한계

- SQLite 파일은 시연용이다. 재배포, 재시작, 저장소 위치 변경 때 데이터가 사라질 수 있다.
- 입고 사진은 앱 파일시스템에 저장된다. 정식 운영에서는 S3/R2 같은 외부 저장소로 바꿔야 한다.
- 실계좌, 실주소, 전화번호, 사업자 정보는 아직 넣지 않는다.
- 결제, 송장, 통관, 운송장 추적은 아직 붙이지 않는다.

## 시연 전 확인

- 셀러가 회원가입 후 주문을 만들 수 있다.
- 운영자가 관리자 계정으로 로그인할 수 있다.
- 운영자가 입고 사진과 검수 결과를 기록할 수 있다.
- 운영자가 견적을 입력할 수 있다.
- 셀러가 예치금 충전 요청과 출고 요청을 볼 수 있다.
- 운영자가 예치금 승인과 포장단위 입력을 할 수 있다.
