# Phase {{PHASE_NUMBER}} Team Leader Intake

상태: Team Leader 작성 전

이 파일은 Team Leader가 `superpowers:brainstorming`으로 후보 고객 결과 요청을 이해하고, 한 Phase로 진행하기에 너무 크거나 작은지 판단한 뒤 작성한다.

## 1. 입력

- 받은 고객 결과 요청: `00-customer-outcome.md`
- 사용한 Superpowers: `superpowers:brainstorming`

## 2. 이해한 고객 결과

- 고객이 끝내려는 일:
- 고객 결과 요청을 잘못 이해하면 고객에게 생기는 피해:
- 이번 Phase에서 반드시 지켜야 하는 결과:

## 2.1 플랫폼 독립 원칙 확인

- 고객 계약이 입력 방식에 의존하지 않는가:
- 고객 계약이 출력 방식에 의존하지 않는가:
- 고객 계약이 실행 엔진에 의존하지 않는가:
- 입력/출력/실행 엔진이 구현 선택지로 분리되어 있는가:
- 고객 계약 변경이 필요해 PM에게 되돌릴 사항:

## 3. Phase 범위 판단

- 판단: 너무 큼 / 너무 작음 / 적정
- 너무 크다면 PM에게 요청할 Phase 분리:
- 너무 작다면 같은 고객 약속 안에서 함께 묶을 인접 고객 결과:
- 아무 일이나 추가하지 않았는가:
- 고객 약속, 제외 범위, 업무 범위, 만족 기준 변경 여부:
- PM 재조정 필요 여부:

## 4. 위험 점검

- 숨은 위험:
- 이번 Phase 범위를 넘는 유혹:
- 통과해도 실패인 경우:
- PM에게 되돌릴 질문:

## 5. Superpowers 준수 확인

- `superpowers:brainstorming`을 사용했는가:
- `superpowers:writing-plans`로 넘어갈 준비가 됐는가:
- 구현은 Builder 서브에이전트에게 맡겨야 함을 확인했는가:
- 고객 약속, 이번에 안 하는 것, 업무 범위, 만족 기준이 바뀌었는가:
- PM에게 되돌릴 질문:
