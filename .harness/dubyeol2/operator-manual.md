# 물류 운영 매뉴얼

상태: 2026-07-06 기준, 로컬 `main` 반영 완료 (`7524139`)

이 문서는 지금 제품으로 실제 확인할 수 있는 일을 정리한다. 정식 운영 전환 전 기준판이므로 실주소, 실전화, 실계좌, 사업자정보, 실결제, 외부 배송조회는 운영 기능으로 쓰지 않는다.

## 1. 지금 할 수 있는 일

셀러는 상품과 SKU를 나눠 주문을 접수하고, 입고 증거, SKU별 검수 상태, 견적, 예치금, 출고 요청, 포장단위 상태를 확인할 수 있다.

운영자는 주문을 입고 처리하고, 사진과 외포장 상태를 남기고, 유료 검수 신청 건의 SKU별 입고·하자·검수 결과를 입력할 수 있다. 이후 SKU별 견적을 저장하고, 셀러 예치금 충전 요청을 승인하거나 거절하고, 출고 요청 주문을 BOX-1, BOX-2 같은 포장단위로 나눌 수 있다.

## 2. 셀러 매뉴얼

### 2.1 가입과 로그인

- 회원가입: `/auth/register`
- 로그인: `/auth/login`
- 로그인 후 기본 화면: `/dashboard`

셀러는 자기 계정의 주문, 예치금, 출고 요청만 볼 수 있다. 다른 셀러의 주문, 박스 구성, 예치금 내역은 보이지 않는다.

### 2.2 주문 접수

위치: `/dashboard/orders/new`

1. `상품 링크`와 `상품명`을 입력한다.
2. 상품 안에 SKU를 추가한다.
3. SKU마다 `옵션`과 `수량`을 입력한다.
4. `구매대행` 또는 `배송대행`을 고른다.
5. 수량·외관·하자 확인이 필요하면 `유료 검수 신청`을 체크한다.
6. 요청 메모가 있으면 남긴다.
7. `접수하기`를 누른다.

접수 후 주문은 `접수됨` 상태가 된다. 주문 목록에서는 상품명, 총수량, SKU 개수, 검수 신청 여부, 상태를 확인할 수 있다.

### 2.3 주문과 SKU 상태 확인

위치: `/dashboard/orders`

주문을 열면 다음 정보를 볼 수 있다.

- 입고 전: 아직 중국 창고에 입고되지 않았다는 안내.
- 입고 후: 입고 사진, 외포장 상태, 유료 검수 결과.
- SKU별 작업 현황: 주문 수량, 입고 수량, 부족 수량, 하자 수량, 검수 상태, 검수 메모.
- 견적 완료 후: SKU별 상품가, 중국배송비, 검수비, 국제운임, 수수료, 예상 합계.

검수를 신청하지 않은 주문은 수량 확인이 완료된 것처럼 표시하지 않는다. 기본 제공은 입고 사진과 외포장 이상 안내다.

### 2.4 예치금 충전 요청

위치: `/dashboard/wallet`

1. 충전 요청 금액을 입력한다. 최소 단위는 1,000원이다.
2. 입금자명을 입력한다.
3. 필요하면 메모를 남긴다.
4. `충전 요청`을 누른다.

요청 직후 잔액은 늘어나지 않는다. 운영자가 승인해야 예치금 잔액과 원장에 반영된다. 운영자가 거절하면 잔액은 바뀌지 않고 거절 사유가 남는다.

### 2.5 출고 요청

위치: `/dashboard/orders/{orderId}`

출고 요청은 아래 조건을 모두 만족할 때 가능하다.

- 주문이 입고 완료 상태다.
- 운영자가 견적을 저장했다.
- 셀러의 예치금 잔액이 견적 금액 이상이다.

`출고 요청하고 예치금 차감`을 누르면 견적 금액이 예치금에서 차감되고, 주문은 `출고 요청` 상태가 된다. 이 단계는 로컬 테스트 원장 기준이다. 실입금, PG, 간편결제, 은행 API는 연결되어 있지 않다.

### 2.6 출고관리와 포장단위 확인

위치: `/dashboard/shipment`

출고 요청한 주문은 출고관리에서 확인한다. 운영자가 포장단위를 저장하면 셀러는 다음 정보를 본다.

- 포장단위 개수.
- 박스 마커: 예 `BOX-1`, `BOX-2`.
- 포장 상태: `포장중`, `포장완료`, `출고대기`.
- 박스별 무게와 CBM.
- 박스 안 SKU 종류와 총 수량.

주문 상세에는 `포장단위 / 패킹리스트 기초` 섹션이 생긴다. 여기서 박스별 SKU 구성, 수량, 무게, CBM, 메모를 확인한다. 정식 PDF 패킹리스트와 실제 배송조회는 아직 연결되어 있지 않다.

## 3. 운영자 매뉴얼

### 3.1 운영자 주문 목록

위치: `/admin/orders`

운영자는 전체 주문을 최신순으로 본다. 상태에 따라 다음 행동이 열린다.

- `접수됨`: `입고 기록`으로 이동한다.
- `입고완료`: 견적이 없으면 `견적 입력`으로 이동한다.
- 견적 완료: 목록에서 견적 완료 상태를 확인한다.

### 3.2 입고 기록

위치: `/admin/orders/{orderId}`

1. 입고 사진을 1~2장 업로드한다.
2. 외포장 이상이 있으면 체크하고 메모를 남긴다.
3. 유료 검수 신청 건이면 SKU별 입고 수량, 하자 수량, 검수 통과 여부, 검수 메모를 입력한다.
4. 검수 미신청 건이면 사진과 외포장 상태만 기록한다.
5. `입고 완료 처리`를 누른다.

입고 완료 후 셀러는 주문 상세에서 입고 사진, 외포장 상태, SKU별 입고·검수 결과를 본다.

### 3.3 견적 입력

위치: `/admin/orders/{orderId}/quote`

1. SKU별 상품 단가를 위안화로 입력한다. 배송대행 건은 상품가 0원이 가능하다.
2. SKU별 중국 내 배송비를 입력한다.
3. 실측 무게 kg을 입력한다.
4. 부피 CBM을 입력한다.
5. 적용 환율을 입력한다.
6. 배송 방식 `해운` 또는 `항공`을 고른다.
7. `견적 저장`을 누른다.

견적 저장 후 셀러 주문 상세에 항목별 견적과 SKU별 견적 근거가 표시된다. 관세·부가세는 통관 시 별도이며, 화면의 합계는 참고용 안내다.

### 3.4 예치금 충전 확인

위치: `/admin/wallet-topups`

운영자는 셀러가 남긴 충전 요청을 확인한다.

- 실제 입금을 확인했으면 `승인`을 누른다.
- 입금자명 불일치 등 문제가 있으면 사유를 입력하고 `거절`을 누른다.

승인하면 셀러 잔액과 예치금 원장에 한 번 반영된다. 거절하면 잔액과 원장은 바뀌지 않는다. 같은 요청을 두 번 처리할 수 없다.

### 3.5 출고 포장 입력

위치: `/admin/shipments`

출고 요청 주문만 출고 처리 목록에 나온다.

1. 주문을 열어 `출고 포장 입력` 화면으로 들어간다.
2. SKU별 출고 가능 수량을 확인한다.
3. 포장단위 마커를 입력한다. 예: `BOX-1`.
4. 포장 상태를 고른다. 예: `포장중`, `포장완료`, `출고대기`.
5. 무게 kg과 부피 CBM을 입력한다.
6. 메모가 있으면 남긴다.
7. 박스에 담을 SKU 수량을 입력한다.
8. `포장단위 저장`을 누른다.

같은 박스 마커로 다시 저장하면 그 박스 구성이 갱신된다. 전체 박스에 배정한 SKU 수량은 출고 가능 수량을 넘을 수 없다.

## 4. 공개 화면과 계산기

비로그인 사용자는 공개 사이트와 정보 페이지를 볼 수 있다.

- 홈: `/`
- 회사 소개: `/about`
- 서비스 안내: `/guide/services`
- 요율 안내: `/guide/pricing`
- 구매대행: `/services/purchase-agency`
- 배송대행: `/services/shipping-agency`
- 스마트오더: `/services/smart-order`
- 출장검품: `/services/inspection`
- 계산기: `/calculators`

계산기는 화면 기준판이다. 실제 통관, 실제 배송조회, 실제 세금 신고, 실제 외부 API 처리는 아직 연결하지 않는다.

## 5. 지금 하면 안 되는 일

다음 기능은 아직 운영 기능이 아니다.

- 정식 운영 전환: 실주소, 실전화, 실계좌, 사업자정보, 약관 본문 확정.
- PG, 간편결제, 은행 API, 자동 입금 확인.
- 1688 검색, 1688 구매 자동화, 1688 OAuth/API.
- 실제 송장 발급, 택배사 API, 세관 API, 외부 배송조회.
- 정식 PDF 패킹리스트 자동 발행.
- 실제 배송 완료 확정.
- deploy, 원격 push, 원격 머지.

## 6. 시스템이 막아주는 기준과 사람이 확인할 기준

아래 기준은 코드와 테스트로 막는다.

- 주문은 상품과 SKU 단위로 저장한다.
- 유료 검수 미신청 건에는 수량·외관·하자 검수 결과를 기록할 수 없다.
- 예치금 충전 요청은 운영자 승인 전까지 잔액에 반영되지 않는다.
- 출고 요청은 입고와 견적이 끝나고 예치금이 충분한 주문만 가능하다.
- 출고 포장단위는 출고 요청 상태 주문에만 붙일 수 있다.
- 박스별 SKU 수량은 출고 가능 수량을 넘을 수 없다.
- 셀러 화면은 `sellerId` 기준으로 자기 주문, 자기 예치금, 자기 포장단위만 보여준다.

아래 기준은 사람이 실제 업무에서 확인해야 한다.

- 입고 사진이 실제 물건 사진인지 확인한다.
- 외포장 이상 메모가 실제 상태와 맞는지 확인한다.
- 유료 검수 결과가 실제 수량, 하자, 외관 상태와 맞는지 확인한다.
- SKU별 상품가, 중국배송비, 무게, CBM, 환율이 실제 견적 근거와 맞는지 확인한다.
- 예치금 충전 승인 전 실제 입금 여부와 입금자명을 확인한다.
- 박스별 SKU 구성, 무게, CBM이 실제 포장 결과와 맞는지 확인한다.

## 7. 현재 완료된 기능 묶음

- Feature 1: 주문 접수, 입고 증거, 기본 입고확인, 유료 검수, 항목별 견적.
- Feature 2: 두리무역 기준 공개 화면과 대시보드 화면.
- Feature 3: 두리무역 1:1 비공개 기준판. 로고, 이미지, 실주소, 전화번호, 계좌, 사업자정보는 제외.
- Feature 4: 견적 후 출고 요청, 예치금 차감, 출고 상태 확인.
- Feature 5: SKU 라인 구조, 예치금 충전 요청/승인, 출고 포장단위, 패킹리스트 기초 데이터.

Phase 8, 즉 우리 회사용 정식 운영 전환은 보류 상태다.
