# Phase 1 Team Leader Intake

상태: Team Leader 작성 완료 (2026-07-04)

이 파일은 Team Leader가 `superpowers:brainstorming`으로 후보 고객 결과 요청을 이해하고, 한 Phase로 진행하기에 너무 크거나 작은지 판단한 뒤 작성한다.

## 1. 입력

- 받은 고객 결과 요청: `00-customer-outcome.md`
- 사용한 Superpowers: `superpowers:brainstorming`

## 2. 이해한 고객 결과

- 고객이 끝내려는 일: 셀러가 주문을 맡기고, 입고 증거(사진·외포장 안내·유료 검수 결과)와 항목별 견적을 화면에서 믿을 수 있게 확인
- 고객 결과 요청을 잘못 이해하면 고객에게 생기는 피해: 검수 무료/유료 경계를 잘못 구현하면 요금 분쟁, 견적 항목 누락이면 숨은비용 불신, 데이터 격리 실패면 유출
- 이번 Phase에서 반드시 지켜야 하는 결과: 입고 증거·견적이 실제와 일치, 검수 미신청 건 "수량 미확인" 명시, 셀러 간 격리

## 2.1 플랫폼 독립 원칙 확인

- 고객 계약이 입력 방식에 의존하지 않는가: 예 (웹 폼은 구현 선택)
- 고객 계약이 출력 방식에 의존하지 않는가: 예 (대시보드는 구현 선택)
- 고객 계약이 실행 엔진에 의존하지 않는가: 예
- 입력/출력/실행 엔진이 구현 선택지로 분리되어 있는가: 예 (00 §1.1)
- 고객 계약 변경이 필요해 PM에게 되돌릴 사항: 없음

## 3. Phase 범위 판단

- 판단: **너무 큼**
- 근거: 코드 0인 신규 프로젝트에서 후보 하나에 ①스캐폴드+랜딩(두리무역 외관) ②셀러 가입/로그인(인증=보안 게이트 대상) ③주문 접수 ④운영자 입고·검수 기록 ⑤요율 엔진+견적 ⑥대시보드+격리가 모두 묶임. 화면 5~6개+역할 2개(셀러/운영자)+요율 엔진 = Machine Check·외부감리 한 사이클에 담기에 과대
- PM에게 요청할 Phase 분리 (각 Phase가 고객이 보는 결과로 끝나는 세로 분할):
  - **Phase 1**: 셀러가 (두리무역 외관의 사이트에서) 가입·로그인하고 주문을 접수하고(유료 검수 선택 포함) 내 주문 목록·상태를 본다 — "주문을 맡길 수 있다"
  - **Phase 2**: 운영자가 입고·확인을 기록하면 셀러가 대시보드에서 입고 증거(사진 1~2장·외포장 안내·유료 검수 결과)를 본다 — "내 물건을 증거로 확인한다"
  - **Phase 3**: 셀러가 항목별 견적(두리무역 요율 초기값: 상품가·수수료 5%+VAT·검수비·예상 국제운임)을 확인한다 — "총비용을 알고 보낼지 결정한다". Feature 1 합격 예시 전체 통과로 기능 완료
- 너무 작다면 함께 묶을 인접 고객 결과: 해당 없음
- 아무 일이나 추가하지 않았는가: 예 — 분할만 했고 범위 추가 없음 (창고 주소/입고ID 발급은 Phase 2 입고의 전제라 Phase 2에 포함)
- 고객 약속, 제외 범위, 업무 범위, 만족 기준 변경 여부: 없음
- PM 재조정 필요 여부: 필요 (Phase 분리 승인)

## 4. 위험 점검

- 숨은 위험: 인증 도입 Phase는 `/security-review` 필수(Machine Check 규칙), 요율의 ¥/₩ 환율 처리 기준 미정(견적 Phase에서 확정 필요), 두리무역 외관 벤치마킹의 저작권 고려(레이아웃·토큰 참고는 가능하나 이미지·카피 원문 복제 금지)
- 이번 Phase 범위를 넘는 유혹: 1688 검색·자동연동, 예치금, 스마트오더 등 두리무역 전체 메뉴 복제 — benchmark는 참고 정본이지 Phase 범위가 아님
- 통과해도 실패인 경우: 00 §3 세 가지 + 셀러/운영자 권한 경계 뚫림
- PM에게 되돌릴 질문: 없음

## 5. Superpowers 준수 확인

- `superpowers:brainstorming`을 사용했는가: 예 (범위 판단·분해)
- `superpowers:writing-plans`로 넘어갈 준비가 됐는가: PM Phase 경계 확정 후 예
- 구현은 Builder 서브에이전트에게 맡겨야 함을 확인했는가: 예 (`superpowers:subagent-driven-development`, 동시 최대 3, TDD)
- 고객 약속, 이번에 안 하는 것, 업무 범위, 만족 기준이 바뀌었는가: 아니오
- PM에게 되돌릴 질문: 없음

## 6. 다음 단계 판정

- `superpowers:writing-plans` 진입 가능 여부: PM이 Phase 1 경계(분할안)를 확정하면 가능
- 근거: 분할은 고객 약속·범위·만족 기준을 바꾸지 않으므로 대표님 재정렬 불필요, PM 확정만 필요
