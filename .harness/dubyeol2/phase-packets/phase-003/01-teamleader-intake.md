# Phase 3 Team Leader Intake

상태: Team Leader 작성 완료 (2026-07-04)

## 1. 입력

- 받은 고객 결과 요청: `00-customer-outcome.md`
- 사용한 Superpowers: `superpowers:brainstorming`

## 2. 이해한 고객 결과

- 고객이 끝내려는 일: 총비용을 항목별로 알고 국제배송 결정
- 잘못 이해하면 생기는 피해: 계산 오류=즉시 돈 문제(가장 민감), 검수비 경계 오인, 숨은 항목
- 반드시 지켜야 하는 결과: 합계=항목 합, 요율표와 계산 일치, 검수 미신청 건 검수비 항목 없음, 면책 문구

## 2.1 플랫폼 독립 원칙 확인

- 입력/출력/실행 엔진 독립: 예 (00 §1.1). PM에게 되돌릴 사항: 없음

## 3. Phase 범위 판단

- 판단: **적정**
- 근거: 운영자 입력 폼·셀러 표시 패턴은 Phase 2에서 확립. 실질 신규는 ①요율 정의 파일(단일 소스, 컷팅 대비) ②견적 계산 순수 함수(TDD 최적 대상) ③입력 폼·표시 카드. Builder Task 3개 규모
- 아무 일이나 추가하지 않았는가: 예 (LCL·결제·환율 API 제외 유지)
- 고객 약속·범위·만족 기준 변경: 없음. PM 재조정 필요: 불요

## 4. 위험 점검

- 숨은 위험: **돈 계산은 부동소수점 금지** — 정수 연산(위안 펀 단위·원 단위) 또는 고정 소수점 필수, 반올림 규칙 명시 필요. 청구중량 공식(부피 CBM→kg 환산 계수 6000 기준: CBM×1,000,000/6000) 검증 필요
- 범위 넘는 유혹: 두리무역 견적 상태 흐름(견적중→견적완료→결제…) 복제 — 이번엔 "견적 표시"만, 상태 전이 없음(RECEIVED에 견적 붙는 구조)
- 통과해도 실패: 00 §3 세 가지
- PM에게 되돌릴 질문: 없음
