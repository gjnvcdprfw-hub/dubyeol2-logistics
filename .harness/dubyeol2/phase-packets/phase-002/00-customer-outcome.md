# Phase 2 고객 결과 요청

상태: PM 후보 작성 완료 (2026-07-04) → Team Leader 범위 판단 대기

이 파일은 PM이 Team Leader에게 넘기는 후보 고객 결과 요청이다. 구현 티켓이 아니라 고객 결과 단위 작업만 적는다.

## 0. L1~L5 정렬 체크

## 1. 연결 맥락

- 연결된 고객 흐름: 흐름 1
- 연결된 메뉴탭 / 화면 영역: 셀러 대시보드(입고 관리) + 운영자 입고 기록 화면 + 창고 주소
- 연결된 기능 약속: Feature 1 (승인 2026-07-04)
- 연결된 Phase: Phase 2 (Feature 1의 2/3, phases.md §2)

## 1.1 플랫폼 독립 원칙 확인

- 고객 계약: 물건이 창고에 도착하면 셀러가 입고 증거(사진 1~2장·외포장 이상 안내, 유료 신청 건은 수량·외관·하자 결과)를 본다
- 이번 Phase의 입력 방식 후보: 운영자 웹 입력 화면(사진 업로드 포함)
- 이번 Phase의 출력 방식 후보: 셀러 웹 대시보드
- 이번 Phase의 실행 엔진 후보: 무관
- 고객 계약에서 제외할 구현 선택지: 특정 스토리지 서비스, WMS 제품, 바코드 스캐너
- 입력/출력/실행 엔진이 바뀌어도 유지되어야 하는 고객 결과: "입고 증거를 셀러가 본다"

## 2. Phase 고객 목표

- 이번 Phase에서 고객이 끝낼 수 있어야 하는 일: 접수한 주문이 "입고 완료"로 바뀌고, 입고 사진 1~2장과 외포장 이상 여부, (유료 검수 신청 건은) 수량·외관·하자 결과를 대시보드에서 확인
- 이 일이 안 되면 고객에게 생기는 피해: 물건이 실제로 들어왔는지 몰라 분실·하자 리스크를 떠안음 — 신뢰 붕괴
- 이번 Phase가 끝나면 고객이 다음으로 할 수 있는 일: 견적 확인(Phase 3) 후 국제배송 결정

## 3. 고객 결과 요청

### 고객 결과 요청 1

- 고객에게 생기는 변화: ①셀러가 창고 주소+내 입고ID를 대시보드에서 복사해 쓸 수 있다(두리무역 벤치: 회원별 입고ID, 원클릭 복사) ②운영자(우리 직원)가 주문 건에 입고 등록(사진 1~2장 업로드·외포장 이상 여부·메모, 유료 검수 신청 건은 수량·외관·하자 결과)을 하면 ③셀러 대시보드에서 해당 주문이 "입고완료" 상태 + 입고 증거로 보인다
- 완료 기준: 합격 예시 — 셀러 A의 "미니가전 보온병 ×100(유료 검수 신청)" 건에 운영자가 사진 2장·외포장 정상·수량 100/100·하자 0을 기록하면, 셀러 A 화면에 그대로 표시. 검수 미신청 건은 사진·외포장 안내만 보이고 "수량 미확인" 유지
- 이번에 안 하는 것: 견적(Phase 3), 출고·배송, 바코드/WMS, 실제 창고 운영 연동, 알림(카카오/이메일)
- 고객 피해 방지 기준: 입고 증거가 실제 기록과 다르게 표시 금지, 타 셀러 주문에 입고 기록 오귀속 금지, 운영자 화면은 셀러 계정으로 접근 불가(권한 분리), 검수 미신청 건에 검수 결과 표시 금지
- 실제 한 건의 QA 예시: 위 완료 기준 + 실패 흐름(수량 97/100·외포장 파손 기록 시 셀러에게 그대로 표시)
- 통과해도 실패인 경우: ①셀러가 운영자 기능에 접근 가능 ②검수 미신청 건이 수량 확인된 것처럼 보임 ③입고 사진이 다른 주문에 표시

## 4. PM 후보 판단

- PM이 판단하고 진행할 수 있는 것: 화면 구성(두리무역 벤치), 상태 이름(입고대기→입고완료), 사진 저장 방식(로컬)
- 대표님 판단이 필요한 것: 없음 (요율·정책 기확정)
- 후보가 너무 클 수 있는 이유: 운영자 역할·권한 분리 + 사진 업로드 + 입고/검수 기록 + 셀러 표시 + 창고주소/입고ID = 신규 역할 1개와 파일 처리가 한 덩어리
- 후보가 너무 작을 수 있는 이유: 없음
- 같은 고객 약속 안에서 묶을 수 있는 인접 고객 결과: 창고 주소/입고ID (이미 포함 — 입고의 전제)
- 절대 묶으면 안 되는 다른 고객 약속: 견적(Phase 3), LCL(Feature 2)
- Team Leader에게 요청하는 범위 판단: 이 후보가 한 Phase로 적정한지. 크면 "창고주소+운영자 입고 기록" / "셀러 입고 증거 표시" 분할 후보 검토
