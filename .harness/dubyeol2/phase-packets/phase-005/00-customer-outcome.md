## 3. 고객 결과 요청

### 고객 결과 요청 1

- 고객에게 생기는 변화: 로그인한 셀러가 두리무역과 같은 대시보드를 본다 —
  - **사이드바 4그룹 12메뉴**: 🏠 마이페이지 / 🛒 구매대행(장바구니·내 상품 리스트·구매대행 관리·스마트오더) / 📦 배송대행(입고 관리·출고 관리·반품 관리·창고 주소) / 📊 서비스(출장검품) / 👤 내정보(내 프로필·예치금 관리)
  - **마이페이지 홈**: 예치금 레드 배너(₩0, 충전 "준비 중") + 현황 카드 5종(구매대행·입고·출고·반품·스마트오더 — 주문/입고는 실데이터 집계, 나머지 0) — 두리무역 상태 칩 그리드
  - **실동작 유지**: 주문 접수·내 주문(→구매대행 관리 메뉴로 연결)·주문 상세(입고 증거·견적)·창고 주소·입고 관리(입고완료 주문 목록 뷰)
  - **화면만("준비 중")**: 장바구니·내 상품·스마트오더·출고·반품·출장검품·예치금 충전 — 각각 두리무역 화면 구조(상태 탭·빈 상태 문구·안내) + 준비 중 명시. 내 프로필은 기본 정보 실표시(이메일·담당자명·회사명) + 나머지 탭 준비 중
- 완료 기준: 12메뉴 순회 404·빈 화면 0, Feature 1 흐름(접수→입고 증거→견적) 회귀 0, 테스트 36 유지, 거짓 기능 0
- 이번에 안 하는 것: 예치금 실충전·장바구니 실동작·출고/반품 실처리, 운영자 화면 개편, 모바일
- 통과해도 실패인 경우: 기존 주문·견적 흐름이 깨지거나, "준비 중" 메뉴가 되는 척 하거나, 두리 카피 복제
