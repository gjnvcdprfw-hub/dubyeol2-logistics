## 3. 고객 결과 요청

### 고객 결과 요청 1 (공개 영역)

- 고객에게 생기는 변화: 비로그인 방문자가 두리무역과 같은 밀도의 공개 사이트를 탐색 — 랜딩(히어로+검색창 자리+서비스 4섹션+플로우+실적+FAQ+CTA+풀 푸터), 상단 9메뉴 전부 진입 가능, 서비스 소개 4종(구매대행·배송대행·스마트오더·검품 — 각각 두리무역 섹션 구성 벤치), 계산기 허브+9종(CBM·용적중량·배송비 실계산, 나머지 "준비 중" UI), 배송조회(UI+준비 중), 회사소개, 가이드 허브, 요금 안내(rates.ts 실값 표), 부가서비스 안내(benchmark 단가표)
- 완료 기준: feature.md 6번 중 공개 영역 — 9메뉴 순회 404·빈 화면 0, 계산기 3종 손검산 일치, 카피 원문 복제 0(grep)
- 고객 피해 방지: 거짓 기능 금지("준비 중" 명시), 두리 카피·실데이터 복제 금지

### 고객 결과 요청 2 (대시보드 영역)

- 고객에게 생기는 변화: 셀러 대시보드가 두리무역 구조(4그룹 12메뉴)로 — 마이페이지(예치금 배너+현황 카드 5종+스마트오더 카드), 기존 기능 메뉴(주문·창고주소) 재배치, 신규 메뉴(장바구니·내상품·스마트오더·출고·반품·출장검품·프로필·예치금)는 두리무역 화면 구조+빈 상태/"준비 중"
- 완료 기준: 12메뉴 순회 404 0, 기존 기능 회귀 0(테스트 32 + Playwright Feature 1 흐름)
