# Phase 4 Team Leader Intake

상태: Team Leader 작성 완료 (2026-07-05)

## 1~2. 입력·이해

- 받은 요청: 00-customer-outcome.md (두리무역 모든 화면). 사용한 Superpowers: `superpowers:brainstorming`
- 반드시 지켜야 하는 결과: 거짓 기능 금지("준비 중" 명시), 두리 카피 원문·실데이터 복제 금지, Feature 1 회귀 0

## 3. Phase 범위 판단

- 판단: **너무 큼** — 화면 약 30개는 한 감리 사이클 과대
- PM에게 요청한 분할: **Phase 4 = 공개 영역 전체**(요청 1: 랜딩 풀+9메뉴+소개 4종+계산기 10+정보 페이지) / **Phase 5 = 대시보드 전체**(요청 2: 12메뉴+예치금 배너+현황 카드)
- 병렬 실행 판단(v2.22 상한 폐지): 공용 셸(헤더·푸터·공통 컴포넌트) 선행 1개 → 이후 화면 그룹 3개(소개 4종/계산기/정보 페이지)는 파일 겹침 없음 → **병렬 Builder 3개**. `superpowers:dispatching-parallel-agents` 적용
- 아무 일이나 추가하지 않았는가: 예. 고객 약속 변경: 없음

## 4. 위험 점검

- 숨은 위험: 카피 표절(두리 원문 그대로 복제) — Builder 프롬프트에 금지 명시+검토에서 grep 대조 / 랜딩 밀도 부족 재발 — 합격 기준을 "섹션 구성 목록"으로 명문화 / 병렬 Builder의 공용 파일 충돌 — 셸 선행으로 회피, 그룹별 디렉토리 분리
- PM에게 되돌릴 질문: 없음
