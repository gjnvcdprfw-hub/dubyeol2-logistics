# Phase 6 고객 결과 요청

상태: PM 재작성·경계 재확정 필요 (2026-07-05)

## 0. 정렬: feature.md(Feature 3) §1~7 — 승인 발화

- "정확하게 두리무역을 그대로 1:1로 복사하는거야, 복사후에 우리회사에 맡게 수정 후 정식 운영 시작할꺼야"
- "로고,이미지,실주소,전화번호,계좌,사업자정보 만빼고 1:1로 만들어줘 디자인, 기능, 구조, 이모티콘 하나까지"
- "응 그래 내 크롬으로 디버깅해서 하나하나 복사해"

## 1. Phase 고객 목표

- 대표님이 정식 운영 전 직접 비교·수정할 수 있도록, 두리무역 공개 영역을 **비공개 기준판**으로 1:1 복사하기 시작한다.
- 이번 Phase 6은 첫 조각이다: **원본 화면·기능 인벤토리 수집 + 공개 화면 1:1 기준판**.
- 제외 6종(로고, 이미지, 실주소, 전화번호, 계좌, 사업자정보)만 자리표시로 대체하고, 디자인·구조·문구·버튼 흐름·상태 문구·아이콘/이모티콘은 같은 역할로 대응시킨다.

## 2. 고객 결과 요청

- 대상: 두리무역 공개 사이트 전체(랜딩, 공개 메뉴, 서비스 소개, 가이드, 계산기, 검색·조회 진입, 약관류 링크 구조)와 그에 대응하는 우리 로컬 공개 화면.
- 방법: 대표님 Chrome에서 두리무역 원본을 열어 화면별로 확인하고, 로컬 기준판과 하나씩 대조한다.
- 완료 기준: 대표님이 원본 공개 화면과 로컬 기준판을 나란히 보았을 때 제외 6종 외 차이를 작업 대상으로 식별할 수 있고, 기준판은 고객에게 노출되지 않는다.
- 통과해도 실패: 복사본이 공개되거나, 제외 6종 실제 값이 코드에 들어가거나, 운영용 수정 전 복사본을 최종본처럼 취급한다.

## 3. 담장

- 로컬/비공개 기준판만 허용. deploy·원격 push·원격 머지·고객 공개 URL 금지.
- 실서비스 상태 변경 금지: 결제, 실입금, 실DB 쓰기, 유료 API 호출 없음.
- 제외 6종의 실제 값은 코드 반입 금지.

## 4. TL 판단

- 기존 `01-teamleader-intake.md`, `02-plan.md`, `03-verification.md`는 "문구 전용 밀착 재작성" 기준 산출물이므로 현재 재정렬 계약에는 맞지 않는다.
- 대표님이 PM 제안에 동의해 Phase 6은 "원본 인벤토리 + 공개 화면 기준판"으로 먼저 진행한다.
- Team Leader는 이 파일 기준으로 범위를 다시 판단한다. 로그인 후 대시보드·돈 흐름·우리회사화는 후속 Phase로 넘긴다.
