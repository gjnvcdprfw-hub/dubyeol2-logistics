# Phase 6 Team Leader Intake

상태: 작성 완료 (2026-07-05). Superpowers: `superpowers:using-superpowers`, `superpowers:brainstorming`

## 1. 입력

- PM 고객 결과 요청: `00-customer-outcome.md`
- 승인된 고객 약속: 두리무역 1:1 비공개 기준판. 로고, 이미지, 실주소, 전화번호, 계좌, 사업자정보만 제외
- 대표님 승인: "응 그래 내 크롬으로 디버깅해서 하나하나 복사해"
- 현재 Phase 6 경계: 원본 화면·기능 인벤토리 수집 + 공개 화면 1:1 기준판

## 2. 범위 판단

- 전체 Feature 3(공개+대시보드+기능 흐름+우리회사화)을 한 Phase로 진행하는 것은 **너무 큼**.
- PM이 제안한 분해(원본 인벤토리 → 공개 화면 → 대시보드/기능 흐름 → 우리회사화)를 대표님이 승인했으므로, Phase 6은 첫 조각으로 적정하다.
- 로그인 후 대시보드, 돈 흐름, 예치금 결제, 우리 회사용 수정은 후속 Phase로 넘긴다.

## 3. Phase 6에서 하는 일

1. 대표님 Chrome에서 두리무역 공개 화면의 메뉴·섹션·CTA·상태 문구·아이콘/이모티콘·링크 구조를 수집한다.
2. 로고·이미지·실주소·전화번호·계좌·사업자정보를 제외 목록으로 표시한다.
3. 우리 로컬 공개 화면을 원본 공개 화면과 나란히 대조 가능한 비공개 기준판으로 맞춘다.
4. 공개 금지 담장과 제외 6종 코드 반입 금지를 검증한다.

## 4. 위험과 대응

- 위험 1: 원본 실제 사업자 정보가 코드에 들어감 → 제외 6종 grep/수동 확인을 Machine Check에 넣는다.
- 위험 2: 로컬 기준판이 운영용처럼 공개됨 → deploy·push·원격 머지 금지, 로컬 URL만 사용한다.
- 위험 3: 공개 화면 작업 중 로그인 후 대시보드 기능까지 섞임 → Phase 6 파일 범위를 공개 영역으로 제한한다.
- 위험 4: 1:1 기준판을 우리회사화 완료본으로 착각함 → `feature.md`와 `03-verification.md`에 "운영 전 수정 필요"를 남긴다.
