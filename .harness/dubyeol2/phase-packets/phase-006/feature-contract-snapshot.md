# Feature 3 계약 스냅샷

상태: Phase 6 Claude 교차감리 기준 보존 (2026-07-05)

이 파일은 Phase 6 교차감리 대기 중 `feature.md`가 다음 Feature로 전환되어도, Phase 6 감리자가 Feature 3 기준을 잃지 않도록 보존한 스냅샷이다.

## Feature 3: 두리무역 1:1 비공개 기준판

### 1. 고객이 하려는 일 (L1)

- 대표님이 정식 운영 전에 비교·수정할 수 있도록, 두리무역을 **비공개 기준판**으로 먼저 1:1 복사한다.
- 이 기준판을 우리 회사에 맞게 수정한 뒤 정식 운영을 시작한다.
- 대표님 발화: "정확하게 두리무역을 그대로 1:1로 복사하는거야, 복사후에 우리회사에 맡게 수정 후 정식 운영 시작할꺼야" (2026-07-05)

### 2. 제품·사업 판단 (L2)

- 기준판은 고객 공개용이 아니라 내부 제작 기준이다.
- 공개 도메인, 검색엔진, 고객 접근 경로에 노출하지 않는다.
- 운영 시작은 1:1 기준판을 우리 회사용 문구·정보·정책으로 바꾼 뒤에만 가능하다.
- 대표님 승인: "a로 잠궈" / "a로해" — 비공개 기준판 선택.

### 3. 범위 (L3)

- **하는 것**: 두리무역의 화면 디자인, 기능 흐름, 구조, 메뉴, 문구, 버튼, 입력폼, 상태 문구, 아이콘/이모티콘 위치와 역할을 1:1로 맞춘다.
- **수집 방식**: 대표님 Chrome으로 두리무역 원본을 직접 열어 화면을 하나씩 확인하고, 로컬 기준판과 대조한다.
- **제외하는 것**: 로고, 이미지, 실주소, 전화번호, 계좌, 사업자정보.
- 제외 6종은 실제 두리무역 값을 넣지 않고 자리표시로 둔다.
- 기존 "문구 재작성" 원칙은 이 feature 안에서는 폐기한다. 단, 정식 운영 전 우리 회사용으로 바꾸는 단계에서 다시 적용한다.

### 4. 절대 생기면 안 되는 일

- 비공개 기준판이 고객에게 공개되는 것.
- 로고, 이미지, 실주소, 전화번호, 계좌, 사업자정보가 코드나 화면에 들어가는 것.
- 기준판 상태에서 정식 운영을 시작하는 것.
- 유료 API, 실입금, 실DB 쓰기, deploy, 원격 push, 원격 머지를 대표님 승인 없이 하는 것.
- 우리 회사용 수정 전 복사본을 최종본처럼 보고하는 것.

### 5. 합격 예시 (L4)

- 대표님이 두리무역 원본과 우리 비공개 기준판을 나란히 봤을 때, 제외 6종만 자리표시이고 나머지 화면 구성·문구 밀도·버튼·기능 흐름·상태 문구·아이콘/이모티콘 위치가 같은 역할로 대응한다.
- 통과해도 실패: 복사본이 고객에게 공개되거나, 제외 6종의 실제 정보가 코드에 들어가거나, "나중에 수정할 기준판"이 아니라 운영용처럼 노출된다.
- 대표님 발화: "로고,이미지,실주소,전화번호,계좌,사업자정보 만빼고 1:1로 만들어줘 디자인, 기능, 구조, 이모티콘 하나까지" (2026-07-05)

### 6. 담장 (L5)

- Chrome 사용: 대표님 Chrome에서 두리무역 원본을 읽고 디버깅·대조한다. 로그인이나 결제가 필요한 실서비스 상태 변경은 하지 않는다.
- 로컬 기준판: 현재 feature 브랜치와 로컬 개발 환경에서만 만든다.
- 공개 금지: deploy, 원격 push, 원격 머지, 검색 노출, 고객 접근 가능한 URL 공개 금지.
- 실데이터 금지: 제외 6종의 실제 값은 수집해도 코드에 넣지 않는다.
- 돈 금지: 실계좌·실입금·유료 API·실DB 쓰기 없음.
