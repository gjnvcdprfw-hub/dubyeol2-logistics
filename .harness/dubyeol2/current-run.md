## 9. 기본 보고 형식

이 형식은 두별2 하네스 진행 보고에만 적용한다. 평상시 일반 대화, 단순 질문 답변, 짧은 확인에는 붙이지 않는다.

```markdown
[담당자] - PM
[Phase n][단계 k/16][Task n][진행자][진행상태]

- 목표: 이번 Phase에서 고객이 ___할 수 있게 한다.
- 지금 하는 일:
  - ___
  - ___
- 확인된 것: ___
- 막힌 것: 없음 또는 ___
- 사용한 Superpowers: ___ (없으면 없음)
- 다음: [Phase n][Task n][진행상태] ___
```

`Phase n`은 프로젝트 전체 누적 Phase 번호다. Task 번호는 각 Phase 안에서 `Task 1`부터 다시 시작한다.

## 10. Superpowers 전체 체크리스트

매 Phase 종료 전 Team Leader가 전체 Superpowers 스킬 목록 기준으로 사용/해당없음/확인불가를 `phase-packets/phase-n/03-verification.md`에 남긴다. PM 완료 보고에는 `사용` 상태인 스킬 목록과 `확인불가` 상태인 스킬 목록을 요약한다. 이 상태판에는 체크리스트를 쌓지 않고, 현재 Phase packet 링크만 둔다.
