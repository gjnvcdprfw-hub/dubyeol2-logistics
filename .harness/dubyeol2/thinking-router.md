# 두별2 사고법 적용 라우터

상태: 두별2 v2.6 사고법 적용 순서 정본

이 문서는 사고법 원칙을 담지 않는다. 현재 상황에서 `thinking.md`의 어떤 구분을 먼저 적용할지 정하는 적용 순서 선택기다.

## 기본 규칙

- PM 정렬, 열린 질문, 기획, 구조 설계, 개선, 방향 제안, 판단 요청에서는 `thinking.md`를 먼저 읽고 이 라우터로 적용 순서를 고른다.
- 단순 사실 답변, 짧은 확인, 명령 실행 보고에는 불필요하게 긴 사고법 출력을 하지 않는다.
- 라우터는 Superpowers workflow를 대체하지 않는다.
- PM 구체화, 대안 비교, 선제 제안은 `superpowers:brainstorming`이 아니다. PM은 Superpowers를 쓰지 않는다.
