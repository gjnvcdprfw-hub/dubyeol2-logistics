## 3. 절대 생기면 안 되는 일

- 두리 카피 원문·이미지·로고·실데이터가 코드에 들어가는 것
- 잔액·차감 계산 오류 (돈 — 정수 연산·이중 기록 방지)
- 결제 없이 구매 진행되거나, 잔액 부족인데 결제되는 것
- 기존 기능 회귀

## 4. 합격 예시 (L4)

- Phase 7: 셀러가 5만원 충전 신청(입금자명 "데모") → 운영자가 입금 확인 → 잔액 ₩50,000 → 견적 ₩442,700 결제 시도 → 잔액 부족 거부 / 50만원 충전 후 결제 → 잔액 ₩107,300·거래 내역 2건·주문 상태 "결제완료"
