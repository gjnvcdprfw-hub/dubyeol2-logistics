# feature.md

상태: 대표님 승인 (2026-07-05) — Feature 2

이 문서는 이번 기능 약속, 즉 고객에게 약속할 변화를 적은 합의서다. 개발은 이 문서가 승인되기 전 시작하지 않는다.

> 제품 비전: 사업자 셀러 대상 중국→한국 통합 물류 플랫폼 — 두리무역(duly.co.kr) 수준 품질 순차 완성. Feature 1(주문 접수→입고 증거→투명 견적)은 2026-07-05 완료·머지. 이 문서는 **Feature 2**를 닫는다. Feature 1 원문은 git 이력(6e94b12 이전)에 보존.

## 0. 연결된 고객 흐름

- 이번 기능이 해결하는 막힘: 사이트가 최소 셸이라 "두리무역 그대로"라는 외관·구조 기준에 못 미침 — 셀러가 완성된 서비스로 신뢰하기 어렵고, 이후 기능이 들어갈 자리(메뉴·화면)가 없음
- 이번 기능이 끝나면: 두리무역의 모든 화면(공개 16 + 대시보드 12메뉴)이 우리 사이트에 존재하고, 이후 기능은 이 껍데기에 채워 넣기만 하면 됨

## 1. 고객이 하려는 일

- 고객이 끝내려는 일: 두리무역을 쓰듯 사이트를 탐색한다 — 서비스 소개를 읽고, 계산기로 비용을 가늠하고, 대시보드에서 모든 메뉴를 본다
- 대표님 발화 근거: "우선 두리의 모든 화면을 구현해" (2026-07-05) / "홈페이지외관은 두리무역 그대로 벤치마킹해줘, 모든 메뉴, 모든기능, 모든 ui" (2026-07-04)

## 2. 고객이 끝났다고 느끼는 순간

- benchmark-duly.md §1 사이트맵의 모든 화면이 우리 사이트에 존재하고, 두리무역과 같은 구조·밀도로 보인다

## 3. 돈 받고 서비스할 때 절대 생기면 안 되는 일

- 없는 기능이 되는 것처럼 보이는 것 — 미구현 동작은 "준비 중"임이 화면에서 명확해야 함 (거짓 기능 금지)
- 두리무역의 카피 원문·이미지·실주소/연락처·상호를 복제하는 것 (구조·레이아웃·밀도만 벤치, 문구는 우리 것)
- Feature 1의 기존 동작(주문·입고·견적·격리·검수 경계)이 깨지는 것

## 4. 이번에 되는 것 (동작 계층 구분)

- **화면 전부**: 공개 — 랜딩(풀 구성), 상단 9메뉴 내비, 1688 상품 검색, 구매대행/배송대행/스마트오더/검품감사 소개 4종, 배송조회, 계산기 허브+9종, 회사소개, 이용 가이드 허브, 요금 안내, 부가서비스 안내 / 대시보드 — 4그룹 12메뉴(마이페이지·장바구니·내상품·구매대행 관리·스마트오더·입고/출고/반품 관리·창고주소·출장검품·프로필·예치금)
- **실동작**: 기존 Feature 1 기능 + 자체 계산 가능한 계산기(CBM·용적중량·배송비 — rates.ts 요율) + 요금 안내(rates.ts에서 표 생성) + 대시보드 현황 카드(실데이터 집계)
- **화면만("준비 중" 명시)**: 1688 실검색, 배송조회 실추적, HS코드/관세/환율/통관 계산기, 스마트오더 업로드, 장바구니·내상품·출고·반품·예치금 충전, 출장검품 신청

## 5. 이번에 안 하는 것

- 외부 연동 일체(1688 API, 물류 추적, 관세청, 환율, 채널톡), 실결제·예치금 충전, LCL 기능(다음 feature), 반응형 모바일 최적화(데스크톱 우선 — 두리무역도 데스크톱 기준 수집)

## 6. 합격 예시

- 셀러가 랜딩에 들어와 상단 메뉴 9개를 다 눌러본다 — 전부 화면이 있고, 두리무역과 같은 구성(예: 배송대행 소개에 자동화 기술·서비스 유형·부가서비스·FAQ 섹션 순서)이다. 배송비 계산기에 12kg/0.08CBM을 넣으면 rates.ts 요율로 해운/항공 비교가 나온다. 대시보드에 로그인하면 12개 메뉴가 다 있고, 예치금 배너(₩0·레드)와 현황 카드 5종이 보인다. 아직 안 되는 메뉴는 "준비 중" 표시가 명확하다.
- 통과해도 실패인 경우: ①미구현 기능이 되는 척(빈 버튼이 조용히 실패) ②두리무역 카피 원문·실데이터 복제 ③기존 기능 회귀(테스트 32개 중 하나라도 깨짐)

## 7. 기능 전체 QA 시나리오

- 정상: 전 메뉴 클릭 순회(공개 9 + 대시보드 12) → 404·빈 화면 0건, 계산기 3종 계산 검증(요율표 손검산), 기존 Feature 1 흐름 회귀 통과
- 실패: "준비 중" 메뉴 진입 시 명확한 안내 화면
- 절대 실패 금지: 격리·검수 경계·견적 정확성 회귀 없음
