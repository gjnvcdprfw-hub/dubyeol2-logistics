## 4. 우리가 두리무역보다 나은 것 (참고)

- **견적 자동 계산**: 두리는 직원 수기 산정 — 우리는 요율 엔진(rates.ts)이 항목별 자동 계산 + 라벨까지 파생 (컷팅 시 일괄 갱신)
- **검수 규칙 강제**: 신청 건 검수 없이 입고완료 불가(감리로 검증) — 두리는 UI 관례에 의존
- **테스트·감리 체계**: 36개 자동 테스트 + 5회 외부감리 PASS

## 5. 예치금 충전 플로우 실물 (2026-07-05 추가 수집 — 세션 복구 후)

**두리무역 충전 = 무통장 입금 + 수동 확인 모델 (PG 없음)** — 우리가 그대로 벤치 가능한 구조:

1. [충전하기] → 모달: 충전 금액(직접 입력 + 프리셋 1·5·10·20·50·100만원, **최소 ₩10,000**)
2. 입금자 정보: 입금자명(필수, **7자 이내**) + 연락처(선택)
3. 입금 계좌 안내: 토스뱅크 계좌+예금주 표시, [복사] 버튼 (실계좌번호는 타사 정보라 기록 생략)
4. 안내사항 5개: ①최소 충전 ₩10,000 ②**출금 불가**(서비스 이용 전용 — 현금 출금 안 됨) ③입금자명 일치 필수 ④**충전 신청 먼저 완료 후 입금** ⑤입금자명 7자 이내
5. [취소]/[충전 요청] → 신청 후 "충전 대기" 탭에 기록 → 운영자가 입금 확인 후 잔액 반영(추정)

→ **다음 기능(예치금) 구현 시 스펙 그대로 사용 가능**: Deposit 모델(신청·대기·확인), 운영자 확인 화면, 잔액 차감은 견적 결제와 연결

### 미수집 잔여

- 구매대행 상세 4탭·출고 신청 폼 실물 (이 계정에 주문 데이터가 없어 진입 불가 — 구조는 가이드 문서로 기수집), 1688 검색 결과·상품 상세(검색 실행이 별도 처리로 미진입)
